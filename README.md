# C# Problem Solving

A curated collection of C# solutions focused on writing clean, maintainable, and efficient code—not just solving problems.

The goal of this repository is to document my journey in improving problem-solving skills while applying software engineering principles such as code readability, reusability, performance optimization, and maintainability.


## Objectives

- Practice algorithmic thinking.
- Write clean and reusable C# code.
- Analyze time and space complexity.
- Apply good software engineering practices.
- Continuously refactor older solutions as my knowledge grows.


## Repository Structure

Each problem has its own folder containing:

- Source code
- Problem description
- Complexity analysis
- Engineering notes
- Possible improvements



## Coding Principles

I try to follow these principles whenever appropriate:

- Clean Code
- Single Responsibility Principle
- Meaningful naming
- Avoid duplicated code
- Prefer reusable methods
- Performance-aware implementations
- Memory-efficient solutions
- Complexity analysis for every solution



## Continuous Improvement

This repository is not intended to store only final answers.

As I learn more advanced software engineering concepts, I revisit older solutions to improve their design, readability, and performance.
